# iOS Setup Guide

This guide will help you build and run the Scripture Scholar iOS app using Capacitor.

## Prerequisites

Before you begin, make sure you have:

1. **macOS** with Xcode installed (available from the Mac App Store)
2. **Node.js** and npm installed
3. **CocoaPods** installed (see below)
4. An **Apple Developer Account** (free or paid)

### Installing CocoaPods

CocoaPods is required to manage iOS dependencies:

```bash
sudo gem install cocoapods
```

If you encounter permission issues:

```bash
gem install --user-install cocoapods
```

Verify installation:

```bash
pod --version
```

## Building the iOS App

### 1. Install Dependencies

First, install all npm dependencies:

```bash
npm install
```

### 2. Build the Web App

Build the React web app to the `dist/` folder:

```bash
npm run build
```

### 3. Sync to iOS

Copy the web app to the iOS project and install CocoaPods dependencies:

```bash
npm run ios:sync
```

This command does two things:
- Copies `dist/` contents to `ios/App/App/public/`
- Installs iOS dependencies via CocoaPods

### 4. Open in Xcode

Open the iOS project in Xcode:

```bash
npm run ios:open
```

Or manually:

```bash
open ios/App/App.xcworkspace
```

⚠️ **Important:** Always open the `.xcworkspace` file, NOT the `.xcodeproj` file!

## Running the App

### On Simulator

1. In Xcode, select a simulator from the device dropdown (e.g., "iPhone 15 Pro")
2. Click the ▶️ Play button or press `Cmd+R`
3. Wait for the app to build and launch

**Note:** PencilKit drawing will NOT work in the simulator. You need a real device.

### On Physical Device

1. Connect your iPad or iPhone via USB
2. In Xcode, select your device from the device dropdown
3. **Configure signing:**
   - Click on the project in the left sidebar
   - Select the "App" target
   - Go to "Signing & Capabilities" tab
   - Check "Automatically manage signing"
   - Select your Team (your Apple ID)
4. Click the ▶️ Play button or press `Cmd+R`

**First time setup:**
- On your device, go to Settings → General → VPN & Device Management
- Tap your developer certificate and tap "Trust"

## Testing PencilKit

PencilKit only works on real iPads/iPhones with Apple Pencil support:

1. Run the app on a physical device
2. Open a Bible chapter
3. Enable annotation mode
4. Tap "Draw with Apple Pencil" button
5. Use Apple Pencil to draw
6. Tap "Done" to save

**Supported devices:**
- iPad Pro (all models)
- iPad Air (3rd gen and later)
- iPad mini (5th gen and later)
- iPad (6th gen and later)

## Development Workflow

### Making Changes to the Web App

```bash
# 1. Make your changes to React components
# 2. Build the web app
npm run build

# 3. Sync to iOS
npm run ios:sync

# 4. Re-run in Xcode (Cmd+R)
```

### Quick Build Script

Use this convenient script for the full workflow:

```bash
npm run build:ios
```

This runs: `npm run build && npx cap sync ios && npx cap open ios`

## Troubleshooting

### CocoaPods Installation Issues

If `sudo gem install cocoapods` fails:

```bash
# Use homebrew instead
brew install cocoapods
```

### Build Errors in Xcode

1. **Clean build folder:** Product → Clean Build Folder (`Cmd+Shift+K`)
2. **Delete derived data:** Go to Xcode → Preferences → Locations → Click arrow next to Derived Data, delete the folder
3. **Reinstall pods:**
   ```bash
   cd ios/App
   rm -rf Pods Podfile.lock
   pod install
   ```

### Signing Issues

- Make sure you're logged into Xcode with your Apple ID (Xcode → Preferences → Accounts)
- For free accounts, you can only install on devices you own
- Paid Developer Program ($99/year) required for App Store distribution

### App Doesn't Launch on Device

- Check that "Automatically manage signing" is enabled
- Trust the developer certificate on the device (Settings → General → VPN & Device Management)
- Try disconnecting and reconnecting the device

### PencilKit Not Working

- PencilKit only works on **real devices** (not simulator)
- Check that the device supports Apple Pencil
- Make sure you tapped "Draw with Apple Pencil" button
- Check Xcode console for error messages

## App Store Submission

To submit to the App Store:

1. **Enroll in Apple Developer Program** ($99/year)
2. **Configure app icons and splash screens** in Xcode
3. **Set version and build numbers** in Xcode
4. **Archive the app:** Product → Archive
5. **Upload to App Store Connect** via Organizer
6. **Fill out App Store metadata** in App Store Connect
7. **Submit for review**

See Apple's [App Distribution Guide](https://developer.apple.com/documentation/xcode/distributing-your-app-for-beta-testing-and-releases) for details.
